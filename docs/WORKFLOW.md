# 工作流说明

## 1. 采集边界

允许采集的公开元数据：

- 榜单名、分类、排名
- 书名、作者名（如公开展示）
- 简介、标签、字数、热度、评分
- 更新时间、连载状态
- 评论关键词的聚合趋势（不保存敏感个人信息）

不建议/不支持：

- 抓取正文、VIP 章节、登录后内容
- 绕过验证码、签名、风控、付费墙
- 批量复制章节再改写
- 模仿某一本书或某个作者的独特表达

## 2. 人审流

```mermaid
flowchart TD
  A[设置题材与约束] --> B[元数据趋势分析]
  B --> C[生成原创选题]
  C --> D[生成大纲 V1]
  D --> E{人工审核}
  E -->|通过| F[导出大纲]
  E -->|修改意见| G[AI 润色大纲]
  G --> E
```

## 3. 质量门禁

- 原创性：不能像具体作品
- 钩子强度：前三章必须有清晰爆点
- 金手指规则：强但有限制
- 冲突密度：每章有目标、阻碍、反转或钩子
- 可连载性：卷纲能支撑 80-150 万字
- 平台适配：题材、节奏、爽点适配目标平台
- 去 AI 味：避免模板套话、空泛总结和同质化对白，用具体物件、动作和人物口吻承载情绪
- 有迹可循反转：先埋低调线索，再改变读者对证据、人物或规则的理解，不能靠空降设定硬拐

## 4. 发布自动化

发布中心把自动化分成三层：

- 本地全自动：生成章节上传包、TXT/Markdown/JSON、发布队列和门禁报告。
- 草稿辅助：输出浏览器助手清单，准备作品、章节标题、正文和作者有话说。
- 人工最终发布：进入番茄作家后台后，最终提交/发布由作者确认。

自动化清单会在登录、验证码、平台风险提示、权限/付费墙、最终发布等节点停止。
